Welcome — Static site deployed on GitHub Pages

This single HTML file contains three "pages" (Home, About, Contact) using client-side hash routing. Edit the content, push to GitHub, then enable Pages in your repo settings.

Why GitHub Pages?

Free hosting for static assets with automatic HTTPS. Great for portfolios, docs, and small marketing sites.

Tips:
  • Use the repository root or the /docs folder as the Pages source.
  • For a custom domain, add a CNAME file and configure DNS.
  • To use Jekyll, add markdown files and _config.yml (Pages supports it automatically).
Example deploy command (replace with your repo):
git add index.html
git commit -m "publish site"
git push origin main

Made for learning

This structure is intentionally simple. When you're ready, split to multiple files or add a build step (Hugo, Eleventy, Vite) and push the generated output to GitHub Pages.
